Kartela stores your tab profiles in your own private storage. Pick a backend and connect to get started.

The token needs only the gist scope. How to create a token

The token needs repository read/write scope. Create one on your instance under Settings → Applications → Generate New Token. Profiles are stored in a private repo named tabula-data (created automatically).

Automatic sync

Kartela normally syncs only when you press a button. These options let a background worker sync for you. Both toggles are independent and may be on at once.

Replace overwrites master with the focused window's tabs every interval — anything only in master is lost.

When on, each timed sync also syncs your bookmarks bar to the backend using the same mode. Your local bookmarks are only ever read, never changed, by the background worker.

When you press Use this profile, this window is first synced into the profile you're leaving (before its tabs are replaced). Merely previewing a profile in the dropdown never syncs. Replace overwrites that outgoing profile's master with the current window.

Migrate data

Copy every profile from one backend to the other. Profiles are copied — the source is never changed, and your local tabs are never touched. Files with the same name on the target are overwritten.

Your token and tabs go only to api.github.com. Kartela has no servers and collects nothing.
